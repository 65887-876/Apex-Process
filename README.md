# Apex Process — Futuristic Onboarding Landing Site

A production-ready, original landing site and multi-step application funnel for
the fictional **Apex Process** brand. Dark, futuristic, glassmorphic UI with a
serious, compliance-first tone — built to convert without overpromising.

> ⚠️ **Originality & legal note.** All copy, branding, sample documents, and
> placeholder testimonials in this repo are **original demonstration content**.
> Nothing here is legal, tax, or financial advice. Have all legal language and
> claims reviewed by qualified counsel before going live. See
> [Legal / compliance notes](#legal--compliance-notes).

---

## Project overview

- A single-page marketing site with a **4-step application wizard**, an
  animated futuristic hero, a 3D-feeling dashboard mockup, a process timeline,
  a placeholder video/testimonial grid, a sample-document center, and a
  compliant FAQ.
- A hardened **`/api/apply`** route: server-side Zod validation, honeypot bot
  protection, IP rate limiting, duplicate-submission protection, structured
  PII-free logging, and TODO hooks for Supabase / Airtable / Resend / Slack /
  CRM.
- Everything is driven by a single config file: **`lib/config.ts`**.

## Tech stack

| Area        | Choice                                            |
| ----------- | ------------------------------------------------- |
| Framework   | Next.js 15 (App Router) + React 19                |
| Language    | TypeScript (strict)                               |
| Styling     | Tailwind CSS                                       |
| Animation   | Framer Motion                                     |
| Forms       | React Hook Form + Zod (`@hookform/resolvers`)     |
| Icons       | lucide-react                                      |
| 3D / FX     | Canvas particle field (no heavy 3D dependency)    |
| Deployment  | Vercel                                            |

> **Why no Three.js / @react-pdf in deps?** To guarantee a fast, dependency-light,
> bulletproof build. The futuristic background is a lazy-loaded `<canvas>` node
> network, and sample PDFs are generated by a zero-dependency Node script. Both
> are easy to swap for React Three Fiber / `@react-pdf/renderer` if you want.

## Environment variables

All variables are documented in **`.env.example`**. None are required for the
site to build and run locally — integrations are optional no-ops until you wire
them up.

| Variable                  | Purpose                                   |
| ------------------------- | ----------------------------------------- |
| `NEXT_PUBLIC_SITE_URL`    | Canonical URL for metadata / Open Graph   |
| `RATE_LIMIT_MAX`          | Max submissions per IP per window         |
| `RATE_LIMIT_WINDOW_MS`    | Rate-limit window in ms                   |
| `SUPABASE_*`              | Supabase insert hook                      |
| `AIRTABLE_*`              | Airtable insert hook                      |
| `RESEND_API_KEY` / `NOTIFY_EMAIL_TO` | Email notifications            |
| `SLACK_WEBHOOK_URL`       | Slack new-application alert               |
| `CRM_WEBHOOK_URL`         | Generic CRM forward                       |

> Secrets are **server-side only**. Never prefix a secret with `NEXT_PUBLIC_`.

## Vercel deployment

1. `npm install` and verify `npm run build` passes locally.
2. Push the repo to GitHub.
3. In Vercel, **Add New → Project** and import the repo.
4. Framework preset auto-detects **Next.js**. No build overrides needed.
5. Add your environment variables under **Settings → Environment Variables**
   (copy keys from `.env.example`).
6. Click **Deploy**.
7. Configure your custom domain; Vercel provisions SSL automatically.

> The in-memory rate limiter resets per serverless cold start. For multi-region
> durability, swap `lib/rate-limit.ts` for Upstash Redis or Vercel KV (TODO
> noted in the file).

## How to edit the brand config

Open **`lib/config.ts`** — it is the single source of truth:

- `brand` — name, tagline, contact emails, URL
- `cta` — button labels
- `flags` — honest, toggleable claims (fee, review window, countdown, gate)
- `timeline` — review-window hours, countdown deadline
- `formOptions` — all dropdown enums (kept in sync with `lib/validation.ts`)
- `trustBadges`, `valueProps`, `whyApex`, `processSteps`
- `faq`, `documents`, `videos`, `socialProof`
- `legal` — all disclaimers and the privacy notice

Form **validation** lives in `lib/validation.ts` and derives its enums from
`formOptions`, so options and validation never drift apart.

## How to replace dummy videos

1. Add real `.mp4` files to **`/public/videos`** (see that folder's README).
2. Update the `videos` array in `lib/config.ts` with `src` and `poster` paths.
3. Add matching raster posters (JPG/PNG) to `/public/images`.

Videos are lazy-loaded — the `<video>` element only mounts after a click — so
placeholders cost nothing. Until replaced, cards show a clearly-labeled
"Placeholder video" state.

## How to replace sample PDFs

Sample PDFs live in **`/public/pdfs`** and are generated by
`scripts/generate-pdfs.mjs` (`npm run generate:pdfs`). To use real documents,
simply drop your own counsel-reviewed PDFs into `/public/pdfs` with the same
filenames referenced in `lib/config.ts` (`documents[].file`), or change those
paths. Keep the visible **SAMPLE DOCUMENT** disclaimer until counsel approves
final versions.

## How to connect Supabase / Airtable / CRM

Open **`app/api/apply/route.ts`** and find the `// TODO(...)` block after
validation. The validated, sanitized `data` object and a generated
`applicationId` are ready to forward. Uncomment / implement the integration you
need:

- **Supabase** — `createClient(...).from("applications").insert(...)`
- **Airtable** — POST to the Airtable REST API
- **Resend** — POST to `https://api.resend.com/emails`
- **Slack** — POST to `SLACK_WEBHOOK_URL` (avoid sending PII to chat)
- **CRM** — POST to `CRM_WEBHOOK_URL`

Add the corresponding env vars in Vercel.

## Legal / compliance notes

This template was built to be **legally cautious by default**:

- **No guarantees.** Copy avoids guaranteed approval / income / return language.
  Honest claims (no fee, 48-hour review) are gated behind `flags` so you can
  turn them off if they aren't true for you.
- **No SSN on the form.** The application never collects Social Security
  numbers; sensitive documents are only *requested* after eligibility review.
- **Consent-first.** The form requires explicit consent checkboxes for review,
  privacy, and a no-guarantee acknowledgment.
- **Sample documents** carry a visible disclaimer and must be reviewed by
  counsel before use.
- **Eligibility logic** (`lib/eligibility.ts`) is placeholder and intentionally
  lenient. Replace it with lawful, non-discriminatory criteria reviewed by
  counsel. Do **not** gate on protected characteristics.
- **Privacy-conscious logging.** The API logs only non-identifying metadata
  (state, referral source, a hashed IP) — never names, email, or phone.

You are responsible for final Terms, Privacy Policy, and Disclosure pages, and
for the accuracy of every claim you publish.

## Architecture

```
app/
  layout.tsx            Root layout, metadata, fonts, ToastProvider, skip link
  page.tsx              Landing page composition
  globals.css           Design system (glass, gradients, reduced-motion)
  api/apply/route.ts    Hardened application submission endpoint
components/
  form/                 ApplicationWizard, Stepper, steps, Success/Ineligible
  sections/             Hero, WhyApex, WhatYouGet, DashboardMockup,
                        ProcessTimeline, ApplySection, VideoGrid,
                        DocumentCenter, FAQ, Navbar, Footer, etc.
  ui/                   Button (magnetic), Field/Input/Select/Checkbox,
                        TiltCard, Reveal, Toast, ParticleField, Logo, icons
lib/
  config.ts             Single source of truth
  validation.ts         Zod schemas (client + server)
  rate-limit.ts         In-memory IP limiter
  application-id.ts     APX-YYYY-MM-XXXX generator
  eligibility.ts        Configurable eligibility gate
  utils.ts              cn, phone/email helpers, sanitizer, hash
public/
  pdfs/                 Generated sample PDFs
  videos/               Drop real videos here
  images/               Posters / generated assets
scripts/
  generate-pdfs.mjs     Zero-dependency PDF generator
```

## Accessibility & performance

- Semantic headings, keyboard-navigable form, ARIA error wiring, visible focus
  rings, a skip link, and `prefers-reduced-motion` support throughout.
- Canvas FX and the video grid are lazy / interaction-gated to protect Core Web
  Vitals. Animations target 60fps and avoid layout shift.
